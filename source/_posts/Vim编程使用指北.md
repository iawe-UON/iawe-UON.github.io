# Vim与IDE的差别：

在使用Vim前，我还是希望搞清楚我们为什么使用Vim进行程序设计，有以下几个方面：

1. Vim避免了频繁的鼠标使用和方向键导肮，反而通过命令行的方式进行快速导航和定位，节约了大量的无效时间
(程序设计的大部分时间都花在了在不同项目间进行导航和代码阅读，连续长时间进行代码编写的几率很少，鼠标和方向键的交互会大大增加时间，而Vim则避免了) 

2. 即便是最基础的Vim也包含了所必须的代码编写的所有功能，而IDE需要第三方联网进行下载、安装，并且IDE的启动速度会比较慢。在面对内网开发环境时，现代IDE基本没有办法进行代码编写 

3. Vim 速度非常快，快就是一切

# Vim常用命令指南：

*强烈建议练中学，在实践中将Vim的各个功能都熟悉一遍，非常轻易的就可以加深印象* 

**Vim的几种状态模式**: 
- **NORMAL MODE**: 默认模式，用于移动和修改，ESC键进入
- **INSERT MODE**: 用于插入内容，单击i键进入
- **VISUAL MODE**: 用于选中内容，单击v键进入
- **SUBSTITUTE MODE**: 用于替换内容 
- **COMMAND MODE**: 用于执行命令，单击":"进入

1. Vim的移动命令：

- 移动到首行：gg，移动到末行：G
- 移动到指定行：冒号:+{行数}或者{行数}+G
- 移动到下一个单词：w，移动到上一个单词：b，移动到下一个单词的开头：e 
- 行操作：移动到行首：0，移动到行尾：$，移动到第一个非空的字母首位：^ 
- 滚动：向上滚动：ctrl+u，向下滚动：ctrl+d 
- 屏幕中的定位：屏幕顶部：H，屏幕底部：L，屏幕中间：M
- 查找搜索：/+{需要查找的内容},使用n/N进行上下文切换

2. Vim的编辑命令：

*在Vim中，编辑命令被称作verb动词，其他命令被称作motion名词，动词作用于名词，这里开始逐渐体现Vim的编程思想*

- 在当前位置进行编辑：i 
- 在当前行前后插入行：1.在当前行后插入空白行：o 2.在当前行前插入空白行：O 
- 删除当前行：dd
- d+{motion}: 删除指定内容,比如：dw表示删除后一个单词，d$表示删除到行尾，d0表示删除到行首
- c+{motion}: 替换指定内容，等价于dw+i
- 进入可视化后选中区块再按d或者c进行删除
- 删除当前字符：x
- 替换当前字符：s
- 撤销：u，撤销当前操作
- 重做：Ctrl+r
- 复制：y，复制指定内容，同样可用可视化来进行框选或者使用y+{motion}进行复制
- 粘贴：p

3. Vim的可视化模式：
- 可视化模式：v
- 可视化行模式： V
- 可视化代码块模式：<C-v>

4. 计数：
- 计数+{verb}: 比如：3dd表示删除3行，3w表示移动3个单词，3x表示删除3个字符

其余同理

5. 修饰符： 

在此处i有另外一个意思，表示在XX内

- ci(:表示替换当前（）内的内容
- ci{:表示替换当前{}内的内容
- da':表示删除单引号引用的字符串，同时也将单引号删除
